#### 1.1 编写后端代码

**`backend/app.py`**

```python
from flask import Flask, jsonify
import redis
import os

app = Flask(__name__)

# 从环境变量读取Redis配置
redis_host = os.getenv('REDIS_HOST', 'localhost')
redis_port = int(os.getenv('REDIS_PORT', 6379))
redis_password = os.getenv('REDIS_PASSWORD', None)

redis_client = redis.Redis(
    host=redis_host,
    port=redis_port,
    password=redis_password,
    decode_responses=True
)

@app.route('/api/ping')
def ping():
    return jsonify({"status": "ok"})

@app.route('/api/set/<key>/<value>')
def set_value(key, value):
    redis_client.set(key, value)
    return jsonify({"key": key, "value": value})

@app.route('/api/get/<key>')
def get_value(key):
    value = redis_client.get(key)
    return jsonify({"key": key, "value": value})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
```

#### 1.3 编写后端Dockerfile

**`backend/Dockerfile.backend`**

```dockerfile
# ── Stage 1: 安装依赖 ─────────────────────────────────
FROM python:3.11-slim AS builder
WORKDIR /build
COPY requirements.txt .
# 将依赖安装到 /build/packages，不污染运行时镜像
RUN pip install --no-cache-dir -r requirements.txt --target /build/packages

# ── Stage 2: 运行时镜像（更小） ──────────────────────
FROM python:3.11-slim
WORKDIR /app
# 只复制依赖包和代码，不包含 pip/编译工具
COPY --from=builder /build/packages /app/packages
COPY . .
ENV PYTHONPATH=/app/packages
EXPOSE 5000
CMD ["python", "app.py"]
```

#### 1.5 编写Nginx配置

**`frontend/nginx.conf`**

```nginx
server {
    listen 80;
    server_name localhost;

    # 静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 反向代理到后端API
    location /api/ {
        proxy_pass http://backend-svc:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 任务4：持久化存储（10分）

**目标**：为Redis配置PVC，确保Pod重建后数据不丢失

#### 4.1 创建PVC

**`k8s/pvc-redis.yaml`**

```yaml
# ── PVC（申请 10Gi 华为云 EVS 云硬盘）──────────────────
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: redis-data-pvc
  namespace: default
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: csi-disk        # 华为云 EVS StorageClass
  resources:
    requests:
      storage: 10Gi
```

#### 4.2 修改Redis Deployment挂载PVC

**`k8s/deployment-redis.yaml`**（更新版）

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: redis
  namespace: default
spec:
  replicas: 1
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
      - name: redis
        image: redis:7-alpine
        ports:
        - containerPort: 6379
        resources:
          limits:
            memory: "512Mi"
        volumeMounts:
        - name: redis-data
          mountPath: /data
      volumes:
      - name: redis-data
        persistentVolumeClaim:
          claimName: redis-data-pvc
```

#### 4.3 部署PVC和更新Redis

```bash
# 创建PVC
```

#### 6.6 报告分析（必须包含）

**1. 扩容延迟原因**：
> HPA扩容存在延迟，主要由两部分组成：
> - metrics-server采集周期：默认15秒采集一次CPU使用率
> - HPA评估间隔：默认15秒评估一次是否需要扩容
> - 因此从负载增加到Pod扩容，至少需要30秒延迟
> - 加上Pod启动时间（拉取镜像、初始化），实际延迟可能更长

**2. 冷却时间意义**：
> HPA设置了冷却时间（stabilization window），默认5分钟，目的是：
> - 防止抖动：避免因短暂的负载波动导致Pod数量频繁变化
> - 保护集群：频繁的扩缩容会给API Server和调度器带来压力
> - 节省成本：避免不必要的Pod创建和销毁

**3. HPA对降本的价值**：
> HPA通过自动调整Pod数量，实现按需使用资源：
> - 空闲时自动缩容到最小副本数（minReplicas），节省计算资源费用
> - 高峰时自动扩容满足需求，保证服务质量
> - 相比固定副本数，可节省30%-70%的资源成本

**Amdahl定律分析**

> **Amdahl定律公式**：S(p) = 1 / ((1-f) + f/p)
> 
> 其中：
> - S(p)：p个处理器的加速比
> - f：可并行化的比例
> - p：处理器数量
> 
> **实测分析**：
> - 假设实测加速比为1.5x（从1个Executor到2个Executor）
> - 根据Amdahl定律反推：1.5 = 1 / ((1-f) + f/2)
> - 解得 f ≈ 0.6（60%可并行化）
> 
> **加速比未达到线性（2x）的原因**：
> 1. **通信开销**：数据在Executor之间传输需要时间
> 2. **序列化/反序列化**：数据在网络传输前需要序列化
> 3. **数据量**：数据量不够大，并行化收益不明显
> 4. **调度开销**：Spark调度任务本身需要时间
> 5. **Shuffle操作**：groupBy需要数据重分布，是主要瓶颈

```python
from mpi4py import MPI
import random

comm = MPI.COMM_WORLD
rank = comm.Get_rank()    # 当前进程编号
size = comm.Get_size()    # 总进程数

N = 10_000_000            # 每个进程各采样 1000 万次

local_count = 0
for _ in range(N):
    x, y = random.random(), random.random()
    if x*x + y*y <= 1.0:
        local_count += 1

# Reduce：将所有进程的 local_count 汇总到 rank=0
total = comm.reduce(local_count, op=MPI.SUM, root=0)

if rank == 0:
    pi = 4.0 * total / (N * size)
    print(f"[{size} processes] π ≈ {pi:.6f}")
```

**题目1：并行矩阵乘法**

**`mpi/serial_matrix_multiply.py`**（串行版）

```python
import numpy as np
import time

def matrix_multiply(A, B):
    """串行矩阵乘法"""
    n = len(A)
    C = [[0]*n for _ in range(n)]
    for i in range(n):
        for j in range(n):
            for k in range(n):
                C[i][j] += A[i][k] * B[k][j]
    return C

# 测试
N = 200  # 矩阵大小
print(f"矩阵大小: {N}x{N}")

A = np.random.rand(N, N).tolist()
B = np.random.rand(N, N).tolist()

start = time.time()
C = matrix_multiply(A, B)
serial_time = time.time() - start

print(f"串行耗时: {serial_time:.2f}s")
print(f"结果验证: C[0][0] = {C[0][0]:.4f}")
```

**`mpi/parallel_matrix_multiply.py`**（MPI并行版）

```python
from mpi4py import MPI
import numpy as np
import time

comm = MPI.COMM_WORLD
rank = comm.Get_rank()
size = comm.Get_size()

N = 200  # 矩阵大小

if rank == 0:
    print(f"矩阵大小: {N}x{N}, 进程数: {size}")
    A = np.random.rand(N, N)
    B = np.random.rand(N, N)
else:
    A = None
    B = np.empty((N, N))

# Broadcast：rank 0 将矩阵 B 广播给所有进程
# 数据流向：rank 0 → 所有其他进程
comm.Bcast(B, root=0)

# Scatter：rank 0 将矩阵 A 按行分发给各进程
# 数据流向：rank 0 的 A[i*local_n:(i+1)*local_n] → 进程 i 的 local_A
local_n = N // size
local_A = np.empty((local_n, N))
comm.Scatter(A, local_A, root=0)

# 本地计算：每个进程计算自己的部分
start = time.time()
local_C = np.dot(local_A, B)

# Gather：各进程将结果收集到 rank 0
# 数据流向：所有进程的 local_C → rank 0 的 C
if rank == 0:
    C = np.empty((N, N))
else:
    C = None
comm.Gather(local_C, C, root=0)

parallel_time = time.time() - start

if rank == 0:
    print(f"并行耗时: {parallel_time:.2f}s")
    print(f"结果验证: C[0][0] = {C[0][0]:.4f}")
```

**题目2：数值积分（梯形法）**

**`mpi/serial_integral.py`**（串行版）

```python
import time

def f(x):
    """被积函数：sin(x)"""
    import math
    return math.sin(x)

def trapezoidal_rule(a, b, n):
    """梯形法数值积分"""
    h = (b - a) / n
    result = (f(a) + f(b)) / 2
    for i in range(1, n):
        result += f(a + i * h)
    result *= h
    return result

# 测试
```

```python
import time
```

**题目3：并行排序（奇偶换序）**

**`mpi/serial_sort.py`**（串行版）

```python
import random
import time

def odd_even_sort(arr):
    """奇偶换序排序"""
    n = len(arr)
    for i in range(n):
        for j in range(i % 2, n - 1, 2):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

# 测试
N = 10000
arr = [random.randint(0, 100000) for _ in range(N)]
print(f"数组大小: {N}")

start = time.time()

serial_time = time.time() - start

print(f"串行耗时: {serial_time:.2f}s")
print(f"排序验证: {sorted_arr == sorted(arr)}")
```

**`mpi/parallel_sort.py`**（MPI并行版）

```python
from mpi4py import MPI
import numpy as np
import time
import random

comm = MPI.COMM_WORLD
rank = comm.Get_rank()
size = comm.Get_size()

N = 10000  # 总数组大小
local_n = N // size

if rank == 0:
    print(f"数组大小: {N}, 进程数: {size}")
    arr = [random.randint(0, 100000) for _ in range(N)]
else:
    arr = None

# Scatter：将数组分发给各进程
local_arr = np.empty(local_n, dtype=int)
comm.Scatter(np.array(arr) if rank == 0 else None, local_arr, root=0)

start = time.time()

# 本地排序
local_arr = sorted(local_arr)

# 奇偶换序：相邻进程比较交换
for phase in range(size):
    if phase % 2 == 0:  # 偶数阶段
        if rank % 2 == 0 and rank < size - 1:
            # 与右边邻居交换
            neighbor = comm.sendrecv(local_arr, dest=rank+1, source=rank+1)
            merged = sorted(list(local_arr) + list(neighbor))
            local_arr = np.array(merged[:local_n])
        elif rank % 2 == 1:
            # 与左边邻居交换
            neighbor = comm.sendrecv(local_arr, dest=rank-1, source=rank-1)
            merged = sorted(list(neighbor) + list(local_arr))
            local_arr = np.array(merged[local_n:])
    else:  # 奇数阶段
        if rank % 2 == 1 and rank < size - 1:
            neighbor = comm.sendrecv(local_arr, dest=rank+1, source=rank+1)
            merged = sorted(list(local_arr) + list(neighbor))
            local_arr = np.array(merged[:local_n])
        elif rank % 2 == 0 and rank > 0:
            neighbor = comm.sendrecv(local_arr, dest=rank-1, source=rank-1)
            merged = sorted(list(neighbor) + list(local_arr))
            local_arr = np.array(merged[local_n:])

# Gather：收集结果
sorted_arr = None
if rank == 0:
    sorted_arr = np.empty(N, dtype=int)
comm.Gather(local_arr, sorted_arr, root=0)

parallel_time = time.time() - start

if rank == 0:
    print(f"并行耗时: {parallel_time:.2f}s")
    print(f"排序验证: {list(sorted_arr) == sorted(arr)}")
```

> **实测与理论差距原因分析**：
>

>    - Broadcast和Gather操作涉及所有进程
>    - 通信时间占比随进程数增加而增大
>
> 2. **进程同步**：
>    - Reduce操作需要等待所有进程完成

>    - 数据初始化、结果收集等串行部分
>    - 根据Amdahl定律，串行部分限制了最大加速比
>
> 5. **缓存效应**：
>    - 分布式计算无法共享缓存
>    - 数据局部性降低

**`mpi/nonblocking_comm.py`**（非阻塞版）

```python
from mpi4py import MPI
import numpy as np
import time

comm = MPI.COMM_WORLD
rank = comm.Get_rank()
size = comm.Get_size()

N = 1000000

if rank == 0:
    data = np.random.rand(N)
else:
    data = np.empty(N)

start = time.time()

# 非阻塞通信：Iscatter立即返回，通信在后台进行
req_scatter = comm.Iscatter(data, None if rank == 0 else data, root=0)

# 可以在通信期间做其他计算（与通信重叠）
# 例如：准备其他数据、执行独立计算
other_work = np.random.rand(100000)
other_result = np.sum(other_work)  # 模拟其他计算

# 等待Scatter通信完成
req_scatter.Wait()

# 本地计算
local_result = np.sum(data)

# 非阻塞通信：Ireduce立即返回
req_reduce = comm.Ireduce(local_result, None, op=MPI.SUM, root=0)

# 可以在通信期间做其他计算
more_work = np.random.rand(100000)
more_result = np.sum(more_work)

# 等待Reduce通信完成
req_reduce.Wait()

nonblocking_time = time.time() - start

if rank == 0:
    print(f"[Non-blocking] 进程数: {size}, 耗时: {nonblocking_time:.4f}s")
    print(f"结果: {local_result if rank == 0 else 'N/A'}")
    print(f"加速: {(blocking_time - nonblocking_time) / blocking_time * 100:.1f}%")
```

**分析说明**

> **非阻塞通信何时有效**：
>
> 1. **计算量大时**：
>    - 当本地计算时间 > 通信时间时，可以充分重叠
>    - 非阻塞通信隐藏了通信延迟
>
> 2. **独立计算多时**：
>    - 如果有与通信结果无关的计算，可以在通信期间执行
>    - 实现计算与通信的真正重叠
>
> 3. **网络延迟高时**：
>    - 在分布式环境中，网络延迟显著
>    - 非阻塞通信可以隐藏这部分延迟
>
> **非阻塞通信何时改善有限**：
>
> 1. **计算量小时**：
>    - 如果本地计算时间 < 通信时间
>    - 等待通信完成的时间没有减少
>
> 2. **数据依赖强时**：
>    - 如果后续计算依赖通信结果
>    - 必须等待通信完成，无法重叠
>
> 3. **网络延迟低时**：
>    - 在同一节点内或高速网络中
>    - 通信本身很快，重叠收益小
>
> **从网络延迟和计算量比例角度**：
> - 设通信时间为 T_comm，计算时间为 T_comp
> - 阻塞版总时间：T_comp + T_comm
> - 非阻塞版总时间：max(T_comp, T_comm)
> - 加速比：(T_comp + T_comm) / max(T_comp, T_comm)
> - 当 T_comp ≈ T_comm 时，加速比最大（接近2x）
> - 当 T_comp << T_comm 时，改善有限

#### C-1 分布式AI训练

**目标**：用Horovod或PyTorch DDP在K8s上训练MNIST CNN

**`ai/train_ddp.py`**

```python
import torch
import torch.nn as nn
import torch.optim as optim
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP
from torch.utils.data import DataLoader, DistributedSampler
from torchvision import datasets, transforms
import time

def setup(rank, world_size):
    dist.init_process_group("nccl", rank=rank, world_size=world_size)

def cleanup():
    dist.destroy_process_group()

class MNISTNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(1, 32, 3, 1)
        self.conv2 = nn.Conv2d(32, 64, 3, 1)
        self.fc1 = nn.Linear(9216, 128)
        self.fc2 = nn.Linear(128, 10)

    def forward(self, x):
        x = torch.relu(self.conv1(x))
        x = torch.relu(self.conv2(x))
        x = torch.max_pool2d(x, 2)
        x = torch.flatten(x, 1)
        x = torch.relu(self.fc1(x))
        x = self.fc2(x)
        return torch.log_softmax(x, dim=1)

def train(rank, world_size):
    setup(rank, world_size)
    
    transform = transforms.Compose([
        transforms.ToTensor(),
        transforms.Normalize((0.1307,), (0.3081,))
    ])
    
    dataset = datasets.MNIST('./data', train=True, download=True, transform=transform)
    sampler = DistributedSampler(dataset, num_replicas=world_size, rank=rank)
    loader = DataLoader(dataset, batch_size=64, sampler=sampler)
    
    model = MNISTNet().to(rank)
    model = DDP(model, device_ids=[rank])
    
    optimizer = optim.Adam(model.parameters())
    
    start = time.time()
    for epoch in range(5):
        model.train()
        for batch_idx, (data, target) in enumerate(loader):
            data, target = data.to(rank), target.to(rank)
            optimizer.zero_grad()
            output = model(data)
            loss = nn.functional.nll_loss(output, target)
            loss.backward()
            optimizer.step()
    
    training_time = time.time() - start
    if rank == 0:
        print(f"Training time: {training_time:.2f}s")
    
    cleanup()

if __name__ == "__main__":
    rank = int(os.environ["RANK"])
    world_size = int(os.environ["WORLD_SIZE"])
    train(rank, world_size)
```

**报告中说明**：
> **AllReduce梯度同步机制**：
> - 每个Worker独立计算梯度
> - AllReduce操作汇总所有Worker的梯度
> - 每个Worker获得相同的平均梯度
> - 使用平均梯度更新模型参数
>
> **数据并行 vs 模型并行**：
> - **数据并行**：每个Worker持有完整模型，处理不同数据子集
>   - 优点：实现简单，扩展性好
>   - 缺点：模型必须能放入单个GPU
> - **模型并行**：模型分布在多个Worker上
>   - 优点：可以训练超大模型
>   - 缺点：通信开销大，负载均衡难

**4. 云端数据接收和存储**

**`edge/cloud_subscriber.py`**

```python
import paho.mqtt.client as mqtt
import redis
import json

BROKER = "localhost"
PORT = 31883
TOPIC = "sensor/temperature"

redis_client = redis.Redis(host='redis-svc', port=6379, decode_responses=True)

def on_message(client, userdata, msg):
    data = json.loads(msg.payload.decode())
    # 存储到Redis
    redis_client.lpush('sensor_data', json.dumps(data))
    # 只保留最近1000条
    redis_client.ltrim('sensor_data', 0, 999)
    print(f"Stored: {data}")

client = mqtt.Client()
client.on_message = on_message
client.connect(BROKER, PORT, 60)
client.subscribe(TOPIC)
client.loop_forever()
```

**报告中说明**：
> **MQTT协议在弱网环境下的适用性**：
> 1. **轻量级**：协议开销小，适合带宽有限的环境
> 2. **发布/订阅模式**：解耦生产者和消费者
> 3. **QoS支持**：提供三种服务质量级别
>    - QoS 0：最多一次，可能丢失
>    - QoS 1：至少一次，可能重复
>    - QoS 2：恰好一次，保证送达
> 4. **遗嘱消息**：设备离线时自动通知
>
> **云边协同延迟挑战**：
> 1. **网络延迟**：边缘到云端的网络延迟（几十到几百毫秒）
> 2. **数据传输**：大量传感器数据上传的带宽压力
> 3. **实时性要求**：某些应用需要毫秒级响应
> 4. **解决方案**：
>    - 边缘预处理：在边缘端过滤、聚合数据
>    - 本地缓存：网络断开时本地存储
>    - 异步通信：非关键数据异步上传

**报告中说明**：
> **Serverless核心概念**：
> 1. **无服务器架构**：开发者无需管理服务器
> 2. **按需付费**：只为实际使用的资源付费
> 3. **自动扩缩容**：根据请求量自动调整资源
> 4. **缩容到零**：空闲时不占用任何资源
>
> **适用场景**：
> 1. **事件驱动**：响应HTTP请求、消息队列等事件
> 2. **突发流量**：流量波动大的应用
> 3. **微服务**：小型、独立的服务
> 4. **API后端**：RESTful API服务
>
> **不适合的场景**：
> 1. **长连接**：WebSocket、长轮询
> 2. **后台任务**：长时间运行的批处理
> 3. **有状态应用**：需要持久化状态的服务
> 4. **冷启动敏感**：对延迟要求极高的应用

## 常见问题FAQ

### Q1: Docker构建失败怎么办？

```bash
# 检查Dockerfile语法
docker build --no-cache -t test .

# 查看详细错误信息
docker build --progress=plain -t test .

# 常见问题：
# 1. 依赖安装失败 → 检查requirements.txt
# 2. 文件找不到 → 检查COPY路径
# 3. 权限问题 → 检查文件权限
```

### Q2: kubectl连接集群失败怎么办？

```bash
# 检查kubeconfig文件
kubectl config view

# 检查当前context
kubectl config current-context

# 切换context
kubectl config use-context <context-name>

# 测试连接
kubectl cluster-info
```

### Q3: Pod一直处于Pending状态怎么办？

```bash
# 查看Pod详情
kubectl describe pod <pod-name>

# 常见原因：
# 1. 资源不足 → 查看Events
# 2. PVC未绑定 → kubectl get pvc
# 3. 镜像拉取失败 → 检查镜像地址
# 4. 节点选择器不匹配 → 检查nodeSelector
```

### Q4: HPA不生效怎么办？

```bash
# 检查HPA状态
kubectl get hpa
kubectl describe hpa backend-hpa

# 检查metrics-server
kubectl top nodes
kubectl top pods

# 常见原因：
# 1. metrics-server未安装
# 2. 资源requests未设置
# 3. CPU使用率未达到阈值
```

### Q5: Spark作业失败怎么办？

```bash
# 查看Driver日志
kubectl logs <driver-pod-name>

# 查看Executor日志
kubectl logs <executor-pod-name>

# 常见原因：
# 1. 镜像不存在 → 检查镜像地址
# 2. 资源不足 → 调整memory/cores配置
# 3. 数据路径错误 → 检查S3路径
```
